# Иерархия датасета VineryMSM

## 1. Главная идея

Данные организованы не по камерам, а по **кусту** (`bush`) — это связующая
сущность. Две камеры дают разные «срезы» об одном и том же кусте:

| Камера | Роль | Что детектит | Модели |
|--------|------|--------------|--------|
| `cam1_bush` | «где я / какой куст» + здоровье лозы | ствол лозы + трек-id (1 ствол = 1 куст), **болезнь лозы** (ствол/рукав/побег) | детектор+трекер ствола лозы, классификатор болезни лозы |
| `cam2_canopy` | «что с кустом» | сорт, болезнь листа, стадия соцветия, % созревания грозди | классификатор сорта, детектор болезни, стадия соцветия, регрессия зрелости |

**Болезнь лозы (cam1) vs болезнь листа (cam2) — разные сущности.** Лоза —
многолетняя древесина (эска, эутипоз, ботриосфериоз); лист — сезонные
милдью/оидиум. Болезнь лозы привязывается к кусту **напрямую** (камера 1 уже
знает track_id куста), без межкамерной синхронизации — в отличие от данных
камеры 2.

Поток камеры 1 даёт «личность» куста во времени, поток камеры 2 — его признаки.
**Синхронизатор** по временному окну склеивает их в одну запись `observation`.

## 2. Логическая иерархия

```
Виноградник (vineyard)
└── Ряд (vine_row)            ← один сорт на ряд
    └── Куст (bush)           ← позиция вдоль ряда, глобальный код V1-R03-B017
        └── Проход (pass)     ← одна сессия съёмки (дата/время проезда тележки)
            └── Наблюдение (observation)   ← синхронизированная запись cam1+cam2
                ├── Кадры cam1 (bbox куста, track_id)
                ├── Кадры cam2 (лист / соцветие / гроздь)
                └── Результаты моделей:
                    ├── болезнь лозы (vine_health)        ← камера 1
                    ├── сорт (variety_prediction)         ← камера 2
                    ├── здоровье листа (leaf_health)      ← камера 2
                    ├── стадия соцветия (inflorescence_stage) ← камера 2
                    ├── зрелость грозди (cluster_ripeness, % — сезонно) ← камера 2
                    └── болезнь грозди (cluster_health — сезонно) ← камера 2
```

## 3. Структура каталогов на диске

```
dataset/
├── raw/                                  # сырые потоковые записи как есть
│   └── <vineyard>/<row>/<pass_YYYYMMDD_HHMM>/
│       ├── cam1_bush.mp4
│       ├── cam2_canopy.mp4
│       └── sync.jsonl                     # временная синхронизация двух потоков
│
├── processed/                            # кадры, уже привязанные к кусту
│   └── <vineyard>/<row>/bush_<NNN>/
│       └── <pass_YYYYMMDD_HHMM>/
│           ├── cam1/frame_000123.jpg
│           ├── cam2/frame_000124.jpg
│           └── meta.json                  # связь куст<->кадры<->наблюдение
│
├── annotations/                         # разметка, отдельно по каждой задаче
│   ├── vine_trunk_detection/               # cam1, детекция ствола лозы (YOLO/COCO)
│   ├── vine_disease/                       # cam1, класс болезни лозы + severity
│   ├── variety_classification/            # cam2, метки сорта на кадр/куст
│   ├── leaf_disease/                      # cam2, класс болезни + severity
│   ├── inflorescence_stage/               # cam2, стадия (BBCH-подобная)
│   ├── cluster_ripeness/                  # cam2, регрессия % (0..100), сезонно
│   └── cluster_disease/                   # cam2, класс болезни грозди + severity, сезонно
│
└── splits/                              # train/val/test
    ├── train.txt
    ├── val.txt
    └── test.txt
```

### Важно для обучения: split по кусту, не по кадру
Кадры одного куста сильно похожи. Если они попадут и в train, и в val —
будет утечка и завышенные метрики. Поэтому `splits/` режутся **по `bush_code`
(а лучше по `vine_row`)**, чтобы все кадры одного куста были в одном сплите.

## 4. Соглашение об именах

```
bush_code      = V{vineyard}-R{row:02d}-B{position:03d}   # V1-R03-B017
pass_dir       = pass_{YYYYMMDD_HHMM}
frame_name     = frame_{global_index:06d}.jpg
```

## 5. meta.json (на каждый куст в проходе)

```json
{
  "bush_code": "V1-R03-B017",
  "vineyard": "V1",
  "row": 3,
  "variety": "cabernet_sauvignon",
  "pass": "pass_20260603_1015",
  "observation_id": 4821,
  "cam1_track_id": 57,
  "t_start": 132.4,
  "t_end": 137.9,
  "frames": {
    "cam1": ["cam1/frame_000123.jpg", "..."],
    "cam2": ["cam2/frame_000124.jpg", "..."]
  },
  "labels": {
    "variety":           {"value": "cabernet_sauvignon", "confidence": 0.97},
    "leaf_health":        {"disease": "mildew", "severity": 0.18, "confidence": 0.88},
    "inflorescence_stage":{"stage": "BBCH-65", "confidence": 0.81},
    "cluster_ripeness":   {"ripeness_pct": 42.0, "brix_estimate": 14.5, "confidence": 0.76}
  }
}
```

`meta.json` — это «контракт» между датасетом и БД: те же поля лежат в таблицах,
так что можно в любой момент пересобрать одно из другого.
