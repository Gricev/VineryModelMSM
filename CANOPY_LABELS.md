# Спецификация меток камеры 2 (лист / соцветие / гроздь)

Контракт разметки cam2: **что** размечаем, **какой геометрией** и **какими кодами**.
Это первоисточник для разметчика и для приёма данных. Коды классов и полей
совпадают с `vinery/db/schema.sql` и `CanopyResult` (`vinery/pipeline/models.py`),
чтобы датасет и БД пересобирались один из другого (как `meta.json` в `DATASET.md`).

> cam2 — **отдельная дорожка** от cam1 (`vine_trunk`). Структуру каталогов и
> анти-утечку (split по ряду) переиспользуем из `INGEST.md`, таксономию — нет.

---

## 1. Ключевая модель: орган = фенофаза, а не отдельные объекты

Соцветие и гроздь — это **один репродуктивный орган в разных фенофазах** (BBCH),
а не два конкурирующих класса. Они **не сосуществуют в одном кадре**:

```
BBCH 53–57   inflorescence  (соцветие появилось/развивается)
BBCH 60–69   inflorescence  (цветение)
BBCH 71–79   cluster        (завязь / молодая гроздь — соцветие СТАЛО гроздью)
BBCH 81–89   cluster        (созревание грозди)
```

Следствия для разметки:
- **партия фото = одна фенофаза** (диктуется датой съёмки). В партии метим тот
  орган, что в ней реально есть;
- один сегментатор с классами `inflorescence`/`cluster` их не путает — они
  временно непересекающиеся;
- это ровно поведение схемы: `inflorescence_stage` заполняется на стадии соцветия,
  `cluster_ripeness` — на стадии грозди; неактуальное поле остаётся `NULL`.

---

## 2. Геометрия: маски, два слоя

Все метки cam2 — **полигональные маски** (сегментация), не боксы. На кадре два слоя:

1. **Слой ОРГАНА** — маска самого объекта (`leaf` / `inflorescence` / `cluster`):
   «где он в кадре». Даёт площадь органа `A_орган`.
2. **Слой ПОРАЖЕНИЯ** — маска(и) больной зоны поверх органа, с кодом болезни:
   «что и насколько поражено». Даёт площадь `A_пораж`.

```
severity = A_пораж / A_орган      (клип в [0..1])
```

Так severity из схемы (`0..1 доля поражённой площади`) считается **из площадей
масок**, а не на глаз. Один орган может нести несколько болезней → по маске на
каждую → severity считается **отдельно по каждому disease_code**.

Здоровый орган: размечается **только слой органа**, слой поражения пуст →
`disease_code = healthy`, `severity = 0`. **Размечать `healthy` обязательно** —
без негативов модель учит «орган = болезнь».

## 3. Таксономия классов

### 3.1 Органы (слой 1, классы сегментации)

| id | класс | когда в кадре | поле-следствие |
|----|-------|----------------|----------------|
| 0 | `leaf` | весь сезон | `leaf_health` |
| 1 | `inflorescence` | BBCH 53–69 | `inflorescence_stage`, `cluster_health`* |
| 2 | `cluster` | BBCH 71–89 | `cluster_ripeness`, `cluster_health` |

\* болезнь соцветия пишется в `cluster_health` — см. §3.3.

### 3.2 Болезни ЛИСТА (`leaf_health.disease_code`)

Канон из схемы. Лист присутствует почти весь сезон, метится независимо от органа.

| disease_code | рус. | визуальный признак (не цвет!) |
|--------------|------|-------------------------------|
| `healthy` | здоров | — |
| `mildew` | милдью (ложная мучн. роса) | «масляные пятна» сверху, белый пушок снизу |
| `oidium` | оидиум (мучн. роса) | серо-белый порошистый налёт |
| `black_rot` | чёрная гниль | бурые пятна с тёмной каймой и пикнидами |

### 3.3 Болезнь РЕПРОДУКТИВНОГО ОРГАНА (`cluster_health.disease_code`)

**Решение:** `cluster_health` = здоровье репродуктивного органа на **обеих**
фенофазах (соцветие И гроздь). Отдельной сущности под болезнь соцветия не вводим;
фенофаза восстанавливается по тому, что в наблюдении заполнено
(`inflorescence_stage` vs `cluster_ripeness`). Severity = доля поражённой
**площади органа** (а не «ягод» — определение расширено, см. §6).

| disease_code | рус. | на какой фенофазе | признак |
|--------------|------|--------------------|---------|
| `healthy` | здоров | обе | — |
| `botrytis` | серая гниль | соцветие и гроздь | серый пушистый мицелий, бурые ягоды/соцветие |
| `mildew` | милдью на соцветии/завязи | в осн. соцветие | побурение и усыхание соцветия |
| `sour_rot` | кислая гниль | гроздь (созревание) | мокрые лопнувшие ягоды, мошкара |
| `berry_oidium` | оидиум ягод | гроздь | серый налёт, растрескивание ягод |

### 3.4 Стадия соцветия (`inflorescence_stage.stage_code`, BBCH)

Атрибут на маске `inflorescence`. Разметчик ставит **ближайшую** стадию:

`BBCH-53` соцветия видны · `BBCH-55` цветки скучены · `BBCH-57` цветки разделены ·
`BBCH-60` первые раскрылись · `BBCH-65` полное цветение · `BBCH-68` конец цветения.

### 3.5 Зрелость грозди (`cluster_ripeness.ripeness_pct`)

Скаляр `0..100` — атрибут на маске `cluster`, не отдельная маска.
**Это не болезнь.** Смена цвета при созревании (верайзон) ≠ патология — метить
зрелость и болезнь раздельно. В пилоте болезней — опционально (можно отложить).

### 3.6 Сорт (`variety`)

**Не размечаем покадрово** — сорт известен из ряда (`vine_row.variety_id`, один
сорт на ряд). При желании используется как кросс-проверка, в пилот не входит.

## 4. Правила разметки и спорные случаи

- **Граница органа.** Маска `cluster`/`inflorescence` — по видимому контуру органа,
  без черешка соседних листьев и фона. Лист — по пластине, без соседних листьев
  в один полигон.
- **Перекрытие/частичная видимость.** Метить видимую часть; сильно скрытый орган
  (<~30% видно) — пропускать, как в cam1 пропускают скрытый листвой ствол.
- **Несколько органов в кадре.** Каждый — отдельный полигон (instance). severity
  считается по каждому отдельно.
- **healthy обязателен.** Здоровые листья/грозди метить наравне с больными.
- **Цвет ≠ болезнь.** Тёмная (чёрный сорт) или краснеющая (верайзон) гроздь сама
  по себе не больна. Признак болезни — текстура/морфология (см. §3), не оттенок.
  Классификатор смотрит на кроп органа, сорт берётся из ряда.
- **Несколько болезней на органе.** Отдельная маска поражения на каждый
  disease_code; severity по каждому свой.
- **Пограничные/сомнительные** — помечать атрибутом `uncertain=true`, не выкидывать.

## 5. Формат, стейджинг, бирки

- **Геометрия экспорта:** YOLO-seg (полигоны, нормированные координаты) либо
  COCO-полигоны. severity и фенофаза считаются на приёме из площадей масок.
- **Стейджинг (по фенофазе И ряду — для анти-утечки):**
  `dataset/to_label/canopy/<phase>/`, имя кадра
  `V1-R03__<phase>__pass01__f0007` (`<phase>` ∈ `flowering`/`cluster`/`leaf`).
  Ряд в имени → split режется по ряду (см. `INGEST.md` §3, `DATASET.md` §split).
- **Готовая разметка** → каталоги задач из `DATASET.md` §3:
  `annotations/leaf_disease/`, `annotations/inflorescence_stage/`,
  `annotations/cluster_disease/`, `annotations/cluster_ripeness/`.
- **Предразметка:** SAM2 даёт маску органа (черновик, как в cam1) → ручная правка
  → слой поражения. Маска подходит для обоих органов.

## 6. Маппинг: метка → БД / CanopyResult

| метка разметчика | таблица БД | поле `CanopyResult` |
|------------------|-----------|---------------------|
| маска `leaf` + болезнь + severity | `leaf_health` | `leaf_health{disease_code,severity}` |
| маска `inflorescence` + `BBCH-xx` | `inflorescence_stage` | `inflorescence_stage{stage_code}` |
| маска `inflorescence`/`cluster` + болезнь | `cluster_health` | `cluster_health{disease_code,severity}` |
| маска `cluster` + `ripeness_pct` | `cluster_ripeness` | `cluster_ripeness{ripeness_pct}` |

**Открытый пункт (требует согласования).** Комментарий `cluster_health.severity`
в `schema.sql` сейчас — «доля поражённых **ягод**». По §3.3 определение
расширяется до «доля поражённой **площади органа**» (покрывает и соцветие).
Это правка комментария/смысла, структура таблицы не меняется. Применить — отдельно.
