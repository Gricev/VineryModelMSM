# Наполнение датасета: детекция ствола лозы (vine_trunk, cam1)

Гайд по добавлению **реальных фото и видео** в датасет детектора ствола и их
предобработке. Инструменты: [`vinery/training/ingest.py`](vinery/training/ingest.py)
(нарезка видео + приём разметки) и [`vinery/training/sam2_prelabel.py`](vinery/training/sam2_prelabel.py)
(SAM2-предразметка фото). Иерархию данных см. в [`DATASET.md`](DATASET.md).

> SAM2 даёт **маски**, а детектору нужны **боксы** → SAM2 работает как
> авто-ПРЕДразметка (маска → описанный прямоугольник), результат **обязательно**
> проверяется руками перед обучением.

## 1. Карта папок: где что лежит

```
dataset/
├── raw/                              ← 1. ОРИГИНАЛЫ (ваши mp4/jpg), храним по рядам
│   └── V1/row_03/                       никогда не трогаются, всё остальное пересобираемо
│       ├── pass01.mp4
│       └── photos/*.jpg
│
├── to_label/vine_trunk/              ← 2. СТЕЙДЖИНГ (создаётся сам)
│      name.jpg + name.txt               нарезанные кадры / предразметка → грузим в CVAT
│      ingest_manifest.csv               провенанс кадров из видео
│
├── annotations/vine_trunk_detection/ ← 3. ГОТОВАЯ РАЗМЕТКА (сюда пишет `import`)
│      images/  labels/  bush_index.csv   все кадры ОДНОЙ кучей + бирки рядов
│
└── yolo/vine_trunk/                  ← 4. СБОРКА ПОД ОБУЧЕНИЕ (train/val/test + data.yaml)
```

Поток в одну сторону: **raw → to_label → (CVAT/Roboflow) → annotations → yolo**.

## 2. Предобработка ВРУЧНУЮ (до запуска инструментов)

Нарезку кадров, JPEG-кодирование, именование и маски→боксы делают инструменты.
Руками нужно только подготовить и сгруппировать сырьё:

**Видео** (1 файл = 1 проезд по 1 ряду):
- обрезать до самого проезда (без разворотов/стоянок);
- одно видео — один ряд; если на видео несколько рядов, разрезать по рядам;
- выкинуть смазанные/тёмные дубли — лучше меньше, но чётких кадров.

**Фото** (отдельные снимки):
- разложить **по рядам**: `raw/V1/row_03/photos/`, `raw/V1/row_04/photos/`
  (флаг `--row` задаётся один на запуск);
- ствол в фокусе, разрешение ≥ ~640px по короткой стороне.

## 3. Бирки рядов: что это и зачем

**Кадры лежат одной кучей** — физически по рядам ничего не делится. «Ряд» — это
лишь **метка-столбец** у каждого кадра в `bush_index.csv`:

```
images/                       ← одна папка, все кадры вперемешку
   V1-R03__pass_01__f0000.jpg
   V1-R07__pass_01__f0000.jpg
bush_index.csv                ← бирки:
   V1-R03__pass_01__f0000, V1-R03
   V1-R07__pass_01__f0000, V1-R07
```

**Зачем.** Кадры одного ряда почти идентичны (тот же ствол, проезд, свет). При
случайном сплите они попадут и в train, и в val → модель «узнаёт» кадр, метрики
врут, а на новом поле модель проваливается. Бирка нужна **ровно одному шагу** —
разрезанию train/val/test: все кадры одного ряда уезжают в **один** сплит.
Модель бирку не видит и учится на всём пуле. Подробнее — [`DATASET.md` §split](DATASET.md).

**Бирки не пишутся вручную.** Вы один раз указываете ряд флагом `--row`, и он
зашивается в имя кадра (`V1-R03__...`), откуда `import` его и достаёт:

```
--row 3  →  имя: V1-R03__pass_01__f0000  →  import: bush_code = V1-R03
```

Для старых фото с неизвестным рядом присвойте хотя бы условный номер — важно лишь,
чтобы кадры **одного физического места** имели одну бирку, а разные места — разные.

## 4. Команды: фото

```powershell
# SAM2 рисует черновые боксы; --interactive = обводить ствол мышью на каждом фото
.venv\Scripts\python.exe -m vinery.training.sam2_prelabel images `
    dataset\raw\V1\row_03\photos --vineyard V1 --row 3 --interactive
# → кадры + черновые .txt в dataset\to_label\vine_trunk\

# Без GUI — промпты из JSON {"IMG_001.jpg": {"box": [x0,y0,x1,y1]}}:
.venv\Scripts\python.exe -m vinery.training.sam2_prelabel images `
    dataset\raw\V1\row_03\photos --vineyard V1 --row 3 --prompts prompts.json
```

## 5. Команды: видео

```powershell
# нарезать кадры на разметку (~3 кадра/сек; --every N — каждый N-й кадр)
.venv\Scripts\python.exe -m vinery.training.ingest extract `
    dataset\raw\V1\row_03\pass01.mp4 --vineyard V1 --row 3 --pass 1 --fps 3
# → кадры в dataset\to_label\vine_trunk\ + ingest_manifest.csv
```

Для видео **протяжку маски делают в CVAT/Roboflow** встроенным SAM2-трекером
(готовый UI: промпт ствола → авто-трек по кадрам → правка → экспорт YOLO). Это
удобнее самописного скрипта. Экспериментальная оффлайн-протяжка есть в
`sam2_prelabel.prelabel_video`, но в проде по видео рекомендуется CVAT.

## 6. Финал: разметка → датасет → обучение

```powershell
# 1) Проверка/правка боксов в CVAT/Roboflow (SAM2 = ЧЕРНОВИК!), экспорт в YOLO.
#    Грузим кадры из dataset\to_label\vine_trunk\, экспортируем обратно.

# 2) Принять размеченное в датасет (дедуплицирует, можно гонять многократно)
.venv\Scripts\python.exe -m vinery.training.ingest import --labels <папка_экспорта_YOLO>
#    кадры в нестандартной папке? добавьте --images <папка_с_jpg>

# 3) Собрать YOLO-датасет (split-by-row, без утечки) и обучить
.venv\Scripts\python.exe -m vinery.training.yolo_dataset --root dataset --group-by row
.venv\Scripts\python.exe -m vinery.training.train_trunk --root dataset --epochs 50
```

## 7. Важные моменты

- **Что считать стволом (главное для качества детекции).** Бокс — это **одревесневший
  ствол/штамб лозы** (вертикальная «личность» куста, выходящая из земли), а НЕ копна
  листвы вокруг. Правила разметки:
  - **один бокс на одну лозу** (1 ствол = 1 куст); не дробить ствол на части и не
    объединять два соседних в один бокс;
  - вести бокс **от земли вверх по древесине**, не растягивая на листья/побеги;
  - в кадрах, где ствол скрыт листвой и не виден — **не размечать** (пустой кадр как
    негативный сэмпл полезнее «бокса по листве»).
  Несогласованная разметка (бокс по листве/низу куста) — главная причина двойных
  детекций и фрагментации `track_id` на инференсе (модель «находит» ствол там, где
  его не видно). Это перевешивает любой тюнинг трекера.
- **SAM2 = черновик.** Боксы всегда проверяйте глазами: маска может зацепить
  лишнее или обрезать ствол.
- **Зависимости.** `extract`/`import` работают на `opencv-python`. `sam2_prelabel`
  требует `torch` + `sam2` и **GPU** (CPU-инференс крайне медленный) — гоняйте на
  GPU-машине. Чистую разметку можно вести и в CVAT/Roboflow без локального SAM2.
  Конвертация маска→бокс (`mask_to_yolo_bbox`) и её тест работают без тяжёлых пакетов.
- **Пополнение датасета** — тот же путь для каждой новой партии: `extract`/`prelabel`
  → CVAT → `import`. `bush_index.csv` дедуплицируется, split-by-row держит val честной.
- **Перенос на другие виноградники.** Чтобы модель работала на чужих полях, важны
  (а) **разнообразие** данных в train (разные шпалеры, свет, сорта, ракурсы, поля) и
  (б) честный сплит на максимально крупной независимой единице. Если виноградников
  несколько — режьте val/test **по винограднику** (целый `V2` в тест), а не только
  по ряду. Привязка к точному кусту на сыром материале не нужна — это работа
  локализатора пайплайна; для обучения детектора достаточно бирки ряда.
